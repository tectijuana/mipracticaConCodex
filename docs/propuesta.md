# Guía de propuesta del microproyecto

> Instrucción: completa este documento en primera persona técnica y con datos concretos.

## 1) Título del proyecto
Escribe un nombre corto, claro y descriptivo.

**Ejemplo:** `cli-control-gastos`

## 2) Problema que resolverás
Describe el problema en 4 a 6 líneas.

- ¿Quién tiene el problema?
- ¿Qué impacto tiene?
- ¿Por qué vale la pena resolverlo?

## 3) Objetivo general
Define un objetivo medible y alcanzable.

**Plantilla sugerida:**
> Desarrollar una herramienta en ___ que permita ___ para ___ en un entorno local.

## 4) Alcance (MVP)
Incluye solo funcionalidades mínimas.

- Funcionalidad 1:
- Funcionalidad 2:
- Funcionalidad opcional (si sobra tiempo):

## 5) Fuera de alcance
Anota explícitamente lo que NO harás para evitar sobrecarga.

- No incluiré interfaz web.
- No integraré APIs externas de pago.
- No usaré base de datos remota.

## 6) Tecnología seleccionada
Marca una sola opción y justifica:
- [ ] ARM64 Assembly
- [ ] C
- [x] Python
- [ ] Bash

Justificación técnica (3 a 5 líneas):

## 7) Cronograma breve
Propón un plan de 3 etapas.

1. Diseño y especificación.
2. Implementación base.
3. Pruebas y ajustes.

## 8) Riesgos y mitigaciones
| Riesgo | Impacto | Mitigación |
|---|---|---|
| Alcance excesivo | Alto | Reducir a 2 funcionalidades clave |
| Falta de pruebas | Medio | Definir casos mínimos desde el día 1 |
| Dependencias innecesarias | Medio | Usar solo librería estándar |
