# Actividad: Diseña y documenta un microproyecto educativo

## Propósito
En esta actividad propondrás un proyecto pequeño y realizable usando **una sola tecnología** de esta lista:
- ARM64 Assembly
- C
- Python
- Bash

El objetivo no es construir un sistema grande, sino demostrar que puedes:
1. Definir un problema concreto.
2. Diseñar una solución mínima viable.
3. Organizar un repositorio profesional y claro.
4. Planear pruebas básicas desde el inicio.

## Entregables obligatorios
Debes completar y ajustar los archivos en la carpeta `docs/` y mantener funcional el ejemplo de `src/main.py`.

- `docs/propuesta.md`
- `docs/caso_de_uso.md`
- `docs/estructura_repositorio.md`
- `docs/plan_de_pruebas.md`
- `src/main.py`
- `scripts/run.sh`
- `tests/test_plan.md`

## Reglas de la actividad
- Alcance pequeño (máximo 1 o 2 funcionalidades principales).
- Sin APIs pagadas.
- Sin bases de datos externas.
- Sin servicios en la nube.
- Sin contenedores.
- Debe poder ejecutarse localmente con herramientas gratuitas.

## Sugerencias de ideas válidas
- Convertidor de unidades por línea de comandos.
- Analizador simple de archivos de texto.
- Generador de reportes básicos en consola.
- Automatizador de tareas locales con Bash.

## Criterios de evaluación sugeridos
1. **Claridad de la propuesta**: objetivo, alcance y límites bien definidos.
2. **Calidad de documentación**: pasos concretos, lenguaje técnico claro.
3. **Estructura del repositorio**: organización coherente y mantenible.
4. **Pruebas**: casos básicos y de error contemplados.
5. **Ejecución**: el script de corrida funciona sin dependencias complejas.

## Cómo iniciar
1. Revisa `docs/propuesta.md` y completa cada sección.
2. Define un caso de uso principal en `docs/caso_de_uso.md`.
3. Ajusta la estructura en `docs/estructura_repositorio.md`.
4. Escribe tu plan de pruebas en `docs/plan_de_pruebas.md`.
5. Modifica `src/main.py` para representar tu idea.
6. Ejecuta `bash scripts/run.sh`.

## Resultado esperado
Al finalizar tendrás una propuesta técnica compacta, bien documentada y lista para evolucionar en iteraciones futuras.
